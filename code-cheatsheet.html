【1. ドキュメント基本構造】
HTML5 雛形セット HTML5の基本構造一式 <!DOCTYPE html> <html lang="ja"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Document</title> <link rel="stylesheet" href="style.css"> </head> <body> </body> </html> 新規HTMLファイル作成時に最初に展開する、必須の全体ボイラープレート。
<meta> セット 文字コードとレスポンシブ設定 <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> `head`内に記述。文字化け防止と、スマホ・タブレットの画面幅最適化に必須。
<title> ウェブページのタイトル定義 <title>ページタイトル</title> ブラウザのタブ名、ブックマーク名、および検索結果のメインリンクとして表示。
<link href="..."> 外部CSSファイルの読み込み <link rel="stylesheet" href="style.css"> `head`タグの中に記述して、作成した外部のCSSデザイン指示書をHTMLと合体させる。
<script src="..."> 外部JavaScriptファイルの読み込み <script src="script.js" defer></script> `head`タグの中に記述。`defer`属性を付与することで、HTMLの読み込みを邪魔せず安全にJSを実行可能。
【2. レイアウト・セマンティクス（意味構造）】
<div> 汎用ブロックコンテナ <div class="container"> </div> 特定の意味を持たせない配置用。CSSレイアウト（Flexbox/Grid）の適用や、JSの操作単位。
<header> セクションまたはページの導入部 <header class="global-header"> </header> サイト最上部の共通ナビゲーションや、記事コンポーネント内のタイトルエリアを構築。
<nav> 主要なナビゲーションリンクの集合 <nav class="navigation"> </nav> グローバルメニュー、パンくずリスト、ページネーションなど「サイト内の移動導線」を囲む。
<main> ドキュメントの主要コンテンツ領域 <main> </main> 各ページ固有の主役となる情報空間。1ページ内に一意（1つだけ）であることが原則。
<section> テーマ性を持った汎用セクション <section class="content-block"> <h2>セクションタイトル</h2> </section> 見出し（h2等）を伴う「論理的な一塊の章」を形成する。単なるデザイン目的の囲みにはdivを使う。
<article> 独立して完結したコンテンツ構造 <article class="post-card"> </article> ブログ、ニュース記事、タイムラインの投稿など「それ単体で他所に移植・配信できる」単位。
<aside> メインから外れた補足・周辺情報 <aside class="sidebar"> </aside> サイドバー、広告枠、メインコンテンツに関連する用語解説やSNSシェアボタンエリア。
<footer> セクションまたはページの最下部 <footer class="global-footer"> <small>&copy; 2026 クリエイター名</small> </footer> 最下部の著作権表記（Copyright）、サイトマップリンク、関連ドキュメントへのリンクを配置。
【3. テキスト・アウトライン】
見出しタグ（h1〜h3） セクションの階層的な見出し <h1>主要タイトル</h1> <h2>中見出し</h2> <h3>小見出し</h3> ドキュメントのアウトライン構造を決定。SEOやスクリーンリーダーがコンテンツを理解する指標。
<p> ひとつのまとまった段落（テキスト） <p>テキストが入ります。</p> 解説文、詳細情報、紹介テキストなど、通常の文章ブロックを展開する。
<br> 強制的な改行（ラインブレイク） 文末です。<br>次行から開始。 デザインや可読性の調整用。段落の間隔を空ける目的の多用は避け、CSSのmarginを使う。
<span> 意味を持たないインラインコンテナ <span class="text-highlight">特定文字</span> 文中の「一部分だけ」の色やフォントをCSSで制御したいときや、JSで文字を動的追記する標的。
<strong> 内容の強い重要性・深刻性の明示 <strong>重要箇所</strong> ブラウザでは原則太字で描画。視覚的強調だけでなく、検索エンジンに重要箇所を伝える。
【4. メディア・実戦リンクの裏技】
<img> 画像リソースのインライン表示 <img src="path.jpg" alt="画像の説明" loading="lazy"> `src`で画像パスを指定、`alt`で代替テキスト（必須）を定義。`loading="lazy"`で遅延読み込み。
<a> (別タブ外部リンク対応) ハイパーリンク（別タブ展開） <a href="url" target="_blank" rel="noopener">リンクテキスト</a> `href`で遷移先を指定。自分のサイトから離脱させずに別タブで開く安全策の `target="_blank"` と `rel="noopener"` をセット。
<a href="tel:"> 実戦技 スマホで「タップしたら電話発信」させる <a href="tel:00000000000">お電話でお問い合わせ</a> 飲食店の予約サイトや、企業のサポート窓口用CVボタンに絶対に必須になるスマホ専用の指定。
<iframe> インラインフレーム（外部埋め込み） <iframe src="外部URL" width="100%" height="400" frameborder="0"></iframe> GoogleマップやYouTube動画、SNSの投稿ウィジェットなどのコンテンツを自サイト内に非同期で綺麗に埋め込む。
【5. リスト・データ構造】
<ul> <li> セット 順序に意味を持たないリスト構造 <ul class="list-group"> <li>リスト項目1</li> <li>リスト項目2</li> <li>リスト項目3</li> </ul> ドロップダウン、サイドメニュー、カード型レイアウトの並列配置など、多くのUI要素の構造土台。
<ol> <li> セット 順序が重要な意味を持つリスト構造 <ol class="steps"> <li>ステップ1</li> <li>ステップ2</li> <li>ステップ3</li> </ol> チュートリアル、購入フロー、人気ランキングなど「1, 2, 3...」とステップを順に追うコンテンツに使用。
【6. テーブル（表データ構造）】
<table> 構築基本セット 二次元の表データの定義コンテナ <table class="data-table"> <thead> <tr> <th>プラン項目</th> <th>内容・価格</th> </tr> </thead> <tbody> <tr> <td>ライトプラン</td> <td>¥980 / 月</td> </tr> <tr> <td>プレミアムプラン</td> <td>¥2,980 / 月</td> </tr> </tbody> </table> マトリクス状のデータ（スペック比較表、料金システム、会社概要等）を表示する標準レイアウト。
【7. ユーザー入力・フォームコンポーネント】
<form> 構成基本セット ユーザー入力・データ送信エリア <form action="/api/submit" method="POST"> <div class="form-group"> <label for="user-name">お名前</label> <input type="text" id="user-name" name="username" placeholder="山田太郎"> </div> <button type="submit">送信する</button> </form> サーバーサイドやJavaScriptのAPIへ、ユーザーが入力したデータを一括送信するための外枠パーツ。
<select> <option> セット 選択肢（ドロップダウンメニュー） <select name="category" id="category-select"> <option value="">選択してください</option> <option value="val1">お問い合わせ</option> <option value="val2">お見積もりのご依頼</option> </select> 限られたスペースで、定義されたリスト（カテゴリ、プラン、都道府県など）からユーザーにデータを選ばせるUI。
<textarea> 複数行のテキスト入力エリア <textarea name="message" id="message" rows="5" placeholder="内容を入力してください"></textarea> お問い合わせの本文、備考欄、ユーザーレビューなど、改行を含む長文テキストの入力を可能にする。
【8. よく使う実戦HTMLパーツ追加】
<picture> <source> 画面幅や画像形式に応じて画像を出し分ける <picture> <source srcset="image.webp" type="image/webp"> <source srcset="image-sp.jpg" media="(max-width: 768px)"> <img src="image-pc.jpg" alt="画像の説明"> </picture> スマホでは軽い画像、PCでは大きい画像を表示したい時に使う。表示速度改善にもつながる。
<details> <summary> JavaScriptなしで開閉できる説明パーツ <details> <summary>詳しく見る</summary> <p>ここに追加説明を書きます。</p> </details> FAQ、補足説明、長い注意書きなどをコンパクトに見せたい時に便利。最初の学習にもかなり優しい。
<figure> <figcaption> 画像や図と説明文をセットで扱う <figure> <img src="sample.jpg" alt="サンプル画像"> <figcaption>画像の説明テキスト</figcaption> </figure> ブログ記事や作品紹介で、画像に説明文を付けたい時に使う。意味構造としても自然。
data-* 属性 HTMLにJS用の情報を持たせる <button data-id="001" data-name="apple"> 商品を見る </button> ボタンごとにIDや名前を持たせ、クリック時にJSで読み取る。タブ、商品一覧、ゲームUIでも使いやすい。
プロパティ（呪文） 一般的な役割・やりたいこと よく使う設定値の例（クリックで一括コピー） 具体的な使われ方（実戦）
【1. 文字・テキストのデザイン】
color 文字の「色」を変える color: #333333; パーツ毎のカラーを文字に塗ったり、全体の文字色を読みやすく整える。
font-size 文字の「大きさ」を変える font-size: 16px; 見出しをガツンと大きくしたり、スマホ用にテキスト全体のスケールを調整する。
font-weight 文字の「太さ」を変える font-weight: bold; 特に強調したい文字や、見出し、ボタンの文字をカチッと太く見せる。
text-align 文字の「配置（行揃え）」を変える text-align: center; タイトル（h2）や、ボタンの中のテキストを一発で「中央揃え」に配置する。
line-height 行と行の「行間（隙間）」を整える line-height: 1.8; デフォルトだと文章がギュッと詰まって読みにくくなるのを防ぐ必須のプロパティ（1.6〜1.8が綺麗）。
text-decoration: none; リンク（aタグ）の「下線」を消す text-decoration: none; メニューバーやオシャレなボタンの中の文字が、いかにも「ハイパーリンク」な下線付きになるのを消し去る。
【2. 背景・色・枠線】
background-color 箱の「背景の色」を変える background-color: #f5f5f7; サイト全体の背景を薄いグレーにしたり、ボタンやカードの背景にパキッと色をつける。
background-image 箱の「背景に画像」を敷く background-image: url("bg.jpg"); background-size: cover; background-position: center; トップ画面の裏側に、綺麗に撮影された写真を全面（引き伸ばしてフィットさせて）表示する。
border 箱の周りに「枠線」をつける border: 1px solid #dddddd; テーブルの罫線や、白背景ボタンのまわりの繊細な囲み線（1pxの細い線など）を作る。
border-radius 箱の角を丸めて「角丸」にする border-radius: 8px; ボタンの角をフワッと丸くして今風のデザインにしたり、`50%`を指定して正方形の写真を綺麗な「正円」にくり抜く。
【3. 余白・サイズ（レイアウトの命）】
padding 箱の「内側の余白」を作る padding: 20px; ボタンの文字のまわりに十分な隙間を作って押しやすくしたり、コンテンツの文字が枠線にピタッとくっつくのを防ぐ。
margin 箱の「外側の余白（隣との距離）」 margin-bottom: 24px; パーツや段落同士がくっつかないように下に距離をあけたり、左右を自動計算にして真ん中に寄せる。
width 箱の「横幅」を指定する width: 100%; max-width: 1200px; サイト全体の最大幅を制限して大画面モニターでの広がりすぎを防止したり、カードの横サイズを固定する。
height 箱の「縦幅」を指定する height: auto; メインビジュアルの高さ（500px等）をがっちり固定したり、文字量に合わせて自動伸縮（auto）させる。
【4. レイアウト・配置（Flexbox & 最重要プロ技）】
display: flex; 中に入っている要素を「横並び」にする display: flex; メニューバーの文字や、メンバーのカードを横一列に並べる。すべての横並び・並列配置の絶対的な土台！
justify-content:
space-between;		 横並びの要素を「両端に等間隔」で広げる justify-content: space-between; 左側に「ロゴ」、右側に「メニューリンク」を綺麗に引き離して配置する、Webサイトヘッダーの完全王道パターン。
justify-content:
center;		 横並びの要素を「まとめて中央寄せ」にする justify-content: center; 複数並んだボタンのグループや、ページネーション（1 2 3...）を画面全体の横方向の真ん中にギュッと集める。
align-items: center; 横並びにした要素の「上下の中心（縦の中央）」を揃える align-items: center; 背の高さが違うロゴ画像とテキストメニューなど、並列したパーツの上下中央を一直線に美しく揃えてガタつきを無くす。
gap 並んだ要素同士の「隙間・間隔」をあける gap: 20px; flexで横並びにしたカードやボタン同士がぴったりくっつくのを防ぐ（marginで個別に指定するより遥かに壊れにくい！）。
margin: 0 auto; 「箱（要素）そのもの」を画面の真ん中に置く margin: 0 auto; width: 90%; max-width: 1200px; /* 💡必ず幅を指定する */ サイト全体のメインの枠組み（コンテナ）を、大画面ディスプレイの左右中央に綺麗に配置するための必須テクニック。
table-layout: fixed; 表（テーブル）の「列の幅を完全に均等」にする table-layout: fixed; width: 100%; `table`タグに指定。マスの中の文字の長さに関係なく、各列の幅をビシッと1/3や1/4に等間隔化して表示崩れを鉄壁ガード。
position 要素をベースの箱に対して「自由な位置」に重ねる /* 親の箱（基準）に指定 */ .parent { position: relative; } /* 重ねたい子要素（ピン留めパーツ）に指定 */ .child { position: absolute; top: -10px; right: 10px; } ブログカードの右上に「New!」というお知らせラベルを浮かせたり、画像の真上に閉じるボタン（×）を重ねて配置する。
【5. アニメーション・ホバー・動き】
transition 色の変化などを「フワッと滑らか」にする transition: all 0.3s ease; ボタンにマウスを乗せた（hover）瞬間に、背景色がパチッと一瞬で変わるのではなく、0.3秒かけてオシャレに変化させる。
opacity 要素の「透明度（透け具合）」を変える opacity: 0.7; マウスが乗ったボタンやリンク画像を少し透けさせて、ユーザーに「今これが押せる状態だよ」と直感的に伝える（hoverとセットの定番）。
【6. マニアック・プロの隠し味いつ使うの！？】
pointer-events: none; マニアック 要素を完全に「すり抜け状態（幽霊化）」にする pointer-events: none; 【いつ使う？】自作ボタンの上に可愛いアイコン画像を重ねて配置した際、その画像の部分だけクリックが吸い取られてボタンが反応しない時。アイコンにこれをかけると「幽霊化」して裏のボタンにクリックが届く。
user-select: none; マニアック 文字の「範囲選択（青いハイライト）」を禁止する user-select: none; 【いつ使う？】JavaScriptで「クリック連打ゲーム」や、何度もカチカチ切り替える「開閉メニュー」を作った際、ユーザーが連打したせいで文字が青く選択されて画面が汚くなるのを防ぐプロのガード技術。
object-fit: cover; マニアック 縦横比がバラバラな画像を、歪ませずに指定サイズにハメる width: 100%; height: 200px; object-fit: cover; 【いつ使う？】ブログ一覧や商品リストの画像エリア（例：高さ200px固定）に写真を並べる時。元画像の比率がバラバラでも、横にビヨーンと伸びたり縮んだりさせず、オシャレに自動中央トリミングしてフィットさせる。
scroll-behavior: smooth; マニアック ページ内リンク（#）のジャンプを「滑らかスクロール」にする html { scroll-behavior: smooth; } 【いつ使う？】画面右下の「↑トップへ戻る」リンクを踏んだ瞬間、パチッと一瞬でワープするのではなく、映画のエンドロールのようにスルスルと滑らかに自動スクロールさせたい時にHTML全体にかける親切設計の1行。
box-sizing: border-box; マニアック 余白（padding）や線を足しても「箱のサイズを変えない」 *, *::before, *::after { box-sizing: border-box; } 【いつ使う？】横幅50%の箱を2つ綺麗に並べた後、内側に余白（padding: 10px）を足したら、全体のサイズが50%+余白分に肥大化して、横並びがバグって下にドカンと落ちてしまった時の救世主（現代の開発では全員最初にかける）。
【7. レスポンシブ・実務でよく使う追加CSS】
@media 画面幅によってCSSを切り替える @media (max-width: 768px) { .container { padding: 12px; } .card-list { flex-direction: column; } } PCでは横並び、スマホでは縦並びにするなど、スマホ対応の中心になる書き方。
display: grid; カードや画像を格子状に並べる .card-list { display: grid; grid-template-columns: repeat(auto-fit, minmax(220px, 1fr)); gap: 20px; } 商品一覧、記事一覧、ギャラリーなどに便利。画面幅に応じて自動で列数が変わる。
clamp() 最小値〜最大値の範囲でサイズを自動調整する font-size: clamp(1.4rem, 4vw, 3rem); 見出しをスマホでは小さく、PCでは大きくする時に便利。メディアクエリを減らせる。
position: sticky; スクロール中に指定位置でくっつく .search-area { position: sticky; top: 0; z-index: 10; } 検索欄やタブを上に残したまま一覧をスクロールさせたい時に使う。カンペ系UIと相性が良い。
overflow-x: auto; はみ出した内容を横スクロールにする .code-box { overflow-x: auto; white-space: pre; } 長いコード、表、URLなどがスマホで画面外にはみ出す時の保険。コード表示ではかなり重要。
aspect-ratio 要素の縦横比を固定する .movie { aspect-ratio: 16 / 9; width: 100%; } YouTube埋め込み、カード画像、サムネイルなどを崩さず表示する時に便利。
コード（JSの命令・呪文） 一般的な役割・意味 具体的な書き方のテンプレート（クリックで一括コピー） 具体的な使われ方（実戦）
【1. 画面の要素を捕まえる（主語）】
document.querySelector() 画面上の要素を「1つだけ」捕まえる const btn = document.querySelector(".my-button"); 「送信ボタン」や「特定の開閉メニュー」など、ピンポイントで動かしたい特定の単一ターゲットを指名・格納する。
document.querySelectorAll() 条件に合う要素を「全部まとめて」捕まえる const tabs = document.querySelectorAll(".tab-item"); 画面内に3つある「タブボタン」や、5つ並んだ「FAQの質問項目」を一括で全部捕まえて、後からまとめて処理するための準備。
【2. 出来事を見張る（動詞）】
.addEventListener() 要素に「何かが起きたら動く」見張り番を置く target.addEventListener("click", () => { // 💡ここにクリックされたときに起きてほしい処理を書く console.log("ターゲットが押されました！"); }); Webサイトにユーザー操作による「動き」をつける時のすべての基本。「ボタンがクリックされたら」「画面がスクロールされたら」を四六時中監視する。
【3. 箱（クラス）を操作してCSSを動かす（一番安全）】
.classList.add() 要素に新しい「クラス（CSSの服）」を着せる element.classList.add("is-active"); クリックされたタブボタンに、あらかじめCSSで作っておいた「目立つ赤色（アクティブ用CSS）」のクラスを一瞬で上書き適用する。
.classList.remove() 要素から「クラス（CSSの服）」を脱がせる element.classList.remove("is-active"); ユーザーが別のメニューを選んだ時、さっきまでアクティブだった古いボタンから「目立つ色クラス」を剥ぎ取って元のデザインに戻す。
.classList.toggle() クラスの着脱を「クリックするたびに交互に切り替える」 element.classList.toggle("is-open"); スマホ用のハンバーガーメニュー等で、ボタンを押すたびにメニューが出たり（クラス追加）消えたり（クラス削除）するスイッチ機能。
.classList.contains() 要素がそのクラスを今持っているか「確認・条件分岐」する if (element.classList.contains("open")) { // 💡すでに開いている場合の処理 } else { // 💡閉まっている場合の処理 } 「今、ポップアップ画面が表示されているかどうか」を判定して、表示されていたら閉じ、閉じられていたら開くといった高度な条件コントロール。
【4. 中身を直接書き換える・操作する】
.textContent 要素の「文字（テキスト）」を直接書き換える element.textContent = "送信中..."; 「送信ボタン」を押した瞬間に、ボタン内の文字を「送信中...」に変更して、ユーザーが何回も連打するのを防ぐ視覚演出。
.style.〇〇 JSから直接「CSS」をピンポイントで1箇所書き換える /* 一瞬で表示する */ element.style.display = "block"; /* 一瞬で非表示（隠す）にする */ element.style.display = "none"; ポップアップの閉じるボタン（×）を押した瞬間に、画面全体を覆う黒いマスクを即座に「非表示（none）」にする時などによく使う。
【5. データの塊をループ処理する（超定番）】
.forEach() 捕まえた複数の要素を「上から順に全員まとめて処理」する // 💡捕まえた全員（配列）に対して、上から1人ずつ処理を実行 elements.forEach(item => { item.addEventListener("click", () => { console.log("誰かがクリックされたよ"); }); }); 「画面内にある5つのタブボタン全員」に対して、一発でまとめてクリックの見張り番（イベントリスナー）を配置して回る、実戦JSの最重要構文。
【6. フォーム・リロード防止のプロ技いつ使うの！？】
e.preventDefault(); マニアック ブラウザの「勝手なページ移動や自動画面リロード」をストップする const form = document.querySelector("form"); form.addEventListener("submit", (e) => { e.preventDefault(); // 💡ブラウザの勝手な画面リロード（更新）を阻止！ // ここに自分のやりたいJSの処理を書く alert("画面を更新せずに、データを送信しました！"); }); 【いつ使う？】フォームの「送信」ボタンを押した際、ブラウザの初期仕様のせいで画面全体が強制リロード（再読み込み）されてしまい、せっかくJavaScriptで書いたアニメーションやトースト通知の処理が一瞬で消えて真っ白になってしまうのを引き止める時に使う必須テクニック。
【7. 実戦で増やしたいJavaScript追加パーツ】
dataset HTMLの data-* 属性をJSで読み取る const btn = document.querySelector("button"); btn.addEventListener("click", () => { console.log(btn.dataset.id); }); 複数のボタンやカードにIDを持たせて、クリックされた対象の情報を取得する。タブや一覧UIで便利。
イベント委任 親要素1つで子要素のクリックをまとめて処理する const list = document.querySelector(".list"); list.addEventListener("click", (e) => { if (e.target.classList.contains("item")) { console.log("項目が押されました"); } }); 後から追加される要素にもクリック処理を効かせたい時に便利。リスト、カード一覧、ゲームの選択肢UIにも使える。
localStorage ブラウザに小さなデータを保存する // 保存 localStorage.setItem("memo", "保存したい文字"); // 読み込み const memo = localStorage.getItem("memo"); メモ、設定、前回入力、ゲームの簡易セーブなどに使える。自作ツールの実用感が一気に上がる。
createElement() JSで新しいHTML要素を作る const card = document.createElement("div"); card.className = "card"; card.textContent = "新しいカード"; document.body.appendChild(card); リストやカードをデータから自動生成したい時に使う。コード量が増えてきたらかなり重要。
fetch() 外部ファイルやAPIからデータを取得する fetch("data.json") .then(response => response.json()) .then(data => { console.log(data); }); JSONファイルからサンプル一覧を読み込む、APIから情報を取るなど、静的ページから一歩進む時に使う。
setInterval() 一定時間ごとに処理を繰り返す setInterval(() => { console.log("1秒ごとに実行"); }, 1000); 時計、カウントダウン、簡単なゲームループなどに使える。止めたい場合は clearInterval とセットで使う。
